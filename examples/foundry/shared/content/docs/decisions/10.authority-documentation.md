## The Problem After Validation

Validation proves demand exists. But in crowded markets, how do founders differentiate?

Landing pages convert. But conversion requires **trust**.

Generic content marketing is commoditized. Anyone can write "top 10 tips" posts.

**The gap:** Properly cited, expert-level content that's hard to replicate.

---

## The Solution: Authority Documentation

Integrate a documentation system that enables:
- **Auto-discovering citation system** — Write inline references, auto-render at bottom
- **MCP resources** — Source documents exposed to AI tools
- **Sources tracking** — Single page showing verification status
- **Proof of concept** — Dark Sky Maharashtra case study

**Key principle:** Optional, not forced. Validation works without it.

---

## Why This Works

### Competitive Moat

Properly cited content creates barriers:
- **Hard to replicate** — Requires domain expertise and source access
- **Builds authority** — Government/academic sources signal credibility
- **Trust → Conversions** — Expert positioning reduces friction

### AI-Ready Architecture

MCP integration future-proofs:
- **Source PDFs accessible to LLMs** — AI tools can verify claims
- **Documentation as data** — Structured for machine consumption
- **Future ecosystem play** — Positioned for AI-driven research tools

### Progressive Adoption

Stays true to "best part is no part":
- **Separate `/docs` directory** — Not coupled to validation system
- **Independent deployment** — Can host separately if needed
- **Zero dependencies** — Works standalone or integrated

## Technical Implementation

### Auto-Discovering References

**Problem solved:** No manual reference listing required.

**Implementation:**
- Nuxt Content transformer extracts all `:reference{}` components
- Custom layout auto-renders references at page bottom
- Zero maintenance — write inline, display automatic

**Developer experience:**
```markdown
Policy established in 2024:reference{id="mpcb-2024" source="MPCB Guidelines" url="/sources/policy.pdf" credibility="government"}
```

References appear automatically. No separate reference section needed.

### MCP Resources

**Problem solved:** Source documents accessible to AI tools.

**Implementation:**
- PDFs stored in `/public/sources`
- MCP resource definitions in `/server/mcp/resources`
- API endpoint `/api/mcp` exposes to AI tools
- AI can retrieve original documents for verification

**Value:** Not just human-readable content. Machine-verifiable sources.

### Smart Cross-Linking

**Problem solved:** Content discoverability and relationship mapping.

**Implementation:**
- **Auto-tracking transformer** — Extracts all internal links `[text](/path)`
- **Bidirectional references** — Shows which pages link to current page
- **Related content component** — Auto-discovers related pages by tags, category, shared links
- **Visual link indicators** — Different icons/colors for policy, research, implementation links
- **Manual override** — Optional `related: [paths]` frontmatter for curated suggestions

**Result:** Content naturally interconnects. Users discover relevant information. Navigation emerges from content structure.

### Sources Tracking

**Problem solved:** Transparency on source credibility.

**Implementation:**
- Auto-generated table from MCP resources
- Shows credibility type (government/academic/NGO)
- Displays verification dates
- Links to original documents

**Result:** Users can verify claims. AI tools can access sources. Trust scales.

## What We Deliberately Excluded

**Will NOT add:**
- ❌ Automated fact-checking (adds complexity)
- ❌ AI-powered source suggestions (premature optimization)
- ❌ Multi-domain templates (keep it general-purpose)
- ❌ Database storage (file-based is simpler)
- ❌ Forced coupling with validation (stays optional)

**Why?** Each addition moves further from "best part is no part."

Authority docs should be **simple enough to understand, powerful enough to differentiate**.

## Decision Rationale

### Why Now?

**Timing:** After establishing validation core, before feature bloat.

**Context:** Template proves validation works. Users ask "what's next after validation?"

**Answer:** Differentiation through authority. Not more validation features.

### Why This Approach?

**Alternatives considered:**
1. **Recommend external docs platforms** — Adds vendor dependency
2. **CMS integration** — Complexity doesn't justify benefit
3. **Static site generator** — Not integrated with Nuxt stack
4. **No documentation support** — Leaves gap in founder journey

**Chosen approach:** Integrated but optional documentation system.

**Rationale:**
- Maintains stack consistency (Nuxt + Nuxt Content)
- Enables MCP integration (source access for AI)
- Stays simple (file-based, no database)
- Remains optional (validation works without it)

## Learning So Far

### What Worked

**Auto-discovery pattern:**
- Eliminates manual reference management
- Developers love "write once, render automatically"
- Zero maintenance overhead

**MCP-first design:**
- Future-proofs for AI ecosystem
- Simple API surface
- Easy to extend

**Optional architecture:**
- Doesn't force complexity on users who don't need it
- Clear separation from validation core
- Independent deployment possible

### What We're Watching

**Adoption signals:**
- Do users actually build authority content?
- Does authority content improve conversion?
- Is MCP integration valuable or just theoretical?

**Complexity creep:**
- Are users requesting database storage?
- Do they want automated source verification?
- Is the "optional" nature clear enough?

## Founder Reflection

This feature could violate "best part is no part."

We're adding documentation infrastructure to a validation template.

**Why it's justified:**
1. **Addresses real gap** — "What's next after validation?" is a common question
2. **Creates moat** — Cited authority content is hard to replicate
3. **Stays optional** — Core validation works without it
4. **Maintains simplicity** — No database, no complex workflows
5. **AI-ready** — MCP integration future-proofs architecture

**Risk:** Feature becomes expected instead of optional.

**Mitigation:** Clear communication that it's for differentiation, not validation.
